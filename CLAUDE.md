## What this is
Omnilingo is a language-learning app — **as easy to start as Duolingo, but deep enough to actually get you fluent.** Multi-language by design (keyed by language code); **Japanese is the first language.** Mission: *"Fluency, made reachable — from your first word to your last."* Built with Expo / React Native (+ react-native-web); ships as a **web PWA on GitHub Pages**.

## How things work (essentials)
- **Content is data, not code.** Lessons live in `data/curriculum/ja/chapterNN/lessonNN.ts` as typed `Lesson` objects; step kinds are defined in `data/curriculum/ja/types.ts` and render automatically. Copy an existing lesson (e.g. `chapter04/lesson01.ts`) as the template.
- **Chapters** are assembled in `data/curriculum/ja/index.ts`; placeholders in `stubs.ts`. Placement test: `data/curriculum/ja/placement.ts` + `scorePlacement` in `data/curriculum/index.ts`.
- **Stroke data** for kana/kanji is real KanjiVG, generated by `node scripts/fetch-strokes.mjs` (extend its char maps + re-run when adding katakana/kanji).
- **Theme**: light/dark palettes in `constants/colors.ts`; `useColors` reads the persisted theme from `AppContext`.

## Ship
- Source of truth = this GitHub repo. Push to `main` → `.github/workflows/web-deploy.yml` builds + deploys to GitHub Pages (~2–3 min). Live: https://anhnqvo-cpu.github.io/Omnilingo/
- **The user pushes via GitHub Desktop** (the git CLI can't authenticate here). So: make commits, then ask them to click Push. Don't try to `git push`.
- The PWA auto-updates (a poller in `scripts/inject-pwa-meta.mjs` reloads on new deploys) — no reinstall needed.
- EAS/OTA workflows (`eas.json`, `RELEASING.md`) exist for the optional TestFlight/App-Store path; web PWA is the primary channel.
